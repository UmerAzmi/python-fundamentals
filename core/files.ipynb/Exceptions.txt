🔹 Common Exception Types in Python
1. SyntaxError
Raised when the Python code itself is written wrong (before running).
# Missing closing parenthesis
print("Hello"
# SyntaxError: unexpected EOF while parsing

2. NameError
When you try to use a variable that hasn’t been defined.
print(x)
# NameError: name 'x' is not defined

3. TypeError
Wrong data type used in an operation.
x = "5"
y = 2
print(x + y)
# TypeError: can only concatenate str (not "int") to str

4. ValueError
Right data type, but invalid value.
num = int("hello")
# ValueError: invalid literal for int() with base 10: 'hello'

5. IndexError
List index is out of range.
lst = [1, 2, 3]
print(lst[5])
# IndexError: list index out of range

6. KeyError
Key not found in a dictionary.
d = {"a": 1, "b": 2}
print(d["c"])
# KeyError: 'c'

7. AttributeError
When an object doesn’t have the attribute/method you try to use.
x = 10
x.append(5)
# AttributeError: 'int' object has no attribute 'append'

8. ZeroDivisionError
Dividing by zero.
print(5 / 0)
# ZeroDivisionError: division by zero

9. ImportError / ModuleNotFoundError
When a module can’t be imported.
import non_existent_module
# ModuleNotFoundError: No module named 'non_existent_module'

10. FileNotFoundError
When a file doesn’t exist.
f = open("not_here.txt")
# FileNotFoundError: [Errno 2] No such file or directory: 'not_here.txt'

11. IndentationError
Wrong indentation in code.
def func():
print("Hi")
# IndentationError: expected an indented block

12. RuntimeError
A generic error when something unexpected happens during execution.

13. StopIteration
Raised when a generator or iterator has no more items.
it = iter([1, 2])
print(next(it))
print(next(it))
print(next(it))
# StopIteration
